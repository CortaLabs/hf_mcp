Hack Forums API v2

hf-mcp

A focused MCP server for read/write-safe Hack Forums workflows. Start local, verify auth wiring, then serve with a bounded, inspectable setup.

Safety posture is fail-closed for concrete writes (`confirm_live=true`). Signature Market writes remain unsupported/unexposed in this release. See export_boundary.md for canonical release-boundary detail.

What It Is

hf-mcp provides a practical local MCP service layer over Hack Forums API v2. It is designed for explicit setup, observable auth flow, and clear operator control.

Install & Setup

  1. Install package

    pip install hf-mcp

  2. Initialize local config

    hf-mcp setup init

  3. Run OAuth bootstrap

    hf-mcp auth bootstrap

  4. Check local health

    hf-mcp doctor

  5. Serve MCP endpoint

    hf-mcp serve

  6. Module entrypoint equivalent

    python -m hf_mcp serve

Hosted OAuth Callback

Use the hosted callback page during OAuth authorization:

https://cortalabs.github.io/hf_mcp/oauth_callback.html

The hosted page forwards approved OAuth params to the fixed local target: http://127.0.0.1:8765/callback
